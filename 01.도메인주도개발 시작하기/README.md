# [BOOK]도메인주도개발 시작하기

도메인 주도 설계(DDD) 핵심 개념 파악 및 JPA 기반 도메인 모델 구현 패턴 학습을 위한 개인 북리뷰 레포지토리입니다.  
이론적 아키텍처 수립부터 이벤트 기반 비동기 처리, CQRS 패턴 적용까지의 학습 내용을 정리합니다.

## 학습 목표
- 도메인 모델 패턴 및 애그리거트 단위 설계 역량 내재화
- 인프라스트럭처(JPA)와 도메인 로직 간 DIP 분리 구현 기술 습득
- 바운디드 컨텍스트 간 통합 및 이벤트 주도 아키텍처(EDA) 이해

## 장별 학습 및 리뷰 현황

### [Chapter 1] 도메인 모델 시작하기
- [ ] 도메인 개념 및 전문가와의 지식 공유 (유비쿼터스 언어)
- [ ] 도메인 모델 패턴 도출 및 엔티티/밸류 타입 특징 분석
- [ ] 도메인 모델 내 set 메서드 지양 이유 및 식별자 생성 전략

### [Chapter 2] 아키텍처 개요
- [ ] 네 개의 영역(표현, 응용, 도메인, 인프라) 역할 정의
- [ ] DIP(의존 역전 원칙) 적용을 통한 계층 구조 아키텍처 개선
- [ ] 애그리거트, 리포지터리 등 도메인 영역 핵심 구성요소 파악

### [Chapter 3] 애그리거트
- [ ] 애그리거트 루트 중심의 도메인 규칙 및 데이터 일관성 유지
- [ ] 트랜잭션 범위 설정 원칙 및 ID를 이용한 애그리거트 참조 방식
- [ ] ID 참조 시의 조회 성능(N+1) 문제 및 팩토리 메서드 활용

### [Chapter 4] 리포지터리와 모델 구현(JPA 중심)
- [ ] 스프링 데이터 JPA 기반 리포지터리 구현 및 모듈 배치
- [ ] 엔티티/밸류 기본 매핑 및 AttributeConverter 활용 패턴
- [ ] 밸류 컬렉션 매핑 방식(별도 테이블 vs 단일 칼럼) 및 영속성 전파(CASCADE)

### [Chapter 5] 스프링 데이터 JPA를 이용한 조회 기능
- [ ] 검색 조건 규격을 위한 Specification(스펙) 디자인 패턴 구현
- [ ] 스펙 조합, 정렬, 페이징 처리 메커니즘 적용
- [ ] 동적 인스턴스 생성 및 하이버네이트 @Subselect 활용 조회 최적화

### [Chapter 6] 응용 서비스와 표현 영역
- [ ] 응용 서비스 역할 정의 (도메인 로직 격리 및 트랜잭션 제어)
- [ ] 응용 서비스 구현 패턴 (크기 제어, 인터페이스 필요성 검토)
- [ ] 표현 영역 인터페이스 및 데이터 검증/권한 검사 레이어 설계

### [Chapter 7] 도메인 서비스
- [ ] 다중 애그리거트 연산 처리를 위한 도메인 서비스 필요성 파악
- [ ] 계산 로직 분리 및 외부 시스템 연동 시 도메인 서비스 활용
- [ ] 도메인 서비스 패키지 배치 전략 및 인터페이스 설계

### [Chapter 8] 애그리거트 트랜잭션 관리
- [ ] 동시성 제어를 위한 선점(Pessimistic) 잠금과 교착 상태 분석
- [ ] 비선점(Optimistic) 잠금 메커니즘 및 강제 버전 증가 패턴
- [ ] LockManager 기반 오프라인 선점 잠금 알고리즘 구현

### [Chapter 9] 도메인 모델과 바운디드 컨텍스트

- [ ] 컨텍스트 간 통합 방식 및 상호 관계 분석 (Shared Kernel, ACL 등)
